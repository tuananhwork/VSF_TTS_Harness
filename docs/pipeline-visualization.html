Scan CODE

uv run python scripts/scan.py

Trích session JSONL từ log raw của Claude Desktop cho 1 ngày. Không gọi LLM — thuần parse + heuristic.

Các bước

  • Discover session theo cây user/workspace/local_*
  • Filter theo ngày (TARGET_DATE, mặc định hôm nay) — giữ session có cửa sổ [createdAt, lastActivityAt] chạm ngày
  • Parse audit.jsonl: dedupe theo uuid, tách turn user / assistant, gom tool_use + tool_result
  • Tín hiệu cấu trúc: gắn cờ repeat (chạy lại cùng tool ở turn sau ≤ 60s) / pivot (user bẻ hướng tool — Jaccard tập tool trước/sau ≥ 0.5). Không keyword, độc lập ngôn ngữ.
  • Compress runs: [A,A,B] → [A×2, B] giữ flow gọn cho judge
  • Tổng hợp tool_usage, tool_sequence, mcp_usage, token, outputs_produced

Output

data/sessions_<date>_runAt_<ts>/
  • <process>__<id>.jsonl — 1 file/session: dòng session_summary + nhiều turn + rate_limit
  • _index.json — scanned / matched + danh sách session

Judge CODE LLM

uv run python scripts/judge.py --sessions-dir …

Triage → recurrence guard → debate đa-judge → candidate skills. Đây là não của pipeline.

Tiền xử lý

  • load_sessions + aggregate → gom thô thành "group" theo tool usage CODE

Pass 1 — Triage LLM

  • Chạy trên summary + tool_sequence + intent_seeds
  • Gom task lặp lại xuyên session, đặt name (kebab-case)
  • Gắn skill_type, trigger_intent (vi/en), evidence, prelim_score

Recompute metric CODE

  • Sau khi LLM merge, tính lại recurrence/repeat_rate/pivot_rate trên evidence thật; loại session_id bịa

Guard — Recurrence CODE

  • Loại candidate có recurrence (đã verify) < --min-recurrence
  • Cap số candidate deep-dive bằng --max-deepdive

Output

data/judge_<date>/
  • cluster_summary.json — các group tool-usage
  • candidate_skills.json — candidate (accepted + rejected)
  • pattern_report.md — báo cáo người đọc
  • _raw_triage / _raw_extract_* / _raw_debate_* / _raw_consolidate_* — debug từng bước

🧩 process_macro

Flow tốt, lặp lại → đóng gói để gọi lại nhanh.

🩹 improvement_lesson

Session lắm repeat/pivot → bài học "lần sau làm X trước".

Pass 2 — Debate (Cách B) · mỗi candidate chạy trên FULL TRACE

Mỗi sub-step lỗi không làm hỏng candidate — lỗi được ghi thành field để consolidator vẫn thấy lỗ hổng.

① Extract LLM

Trung lập, không chấm điểm. Trích action_template (flow có thứ tự), good_points, weak_points, improvement_notes, golden_tests, risk_flags.

→

② Debate LLM

N judge, mỗi judge chấm 1 trục giá trị, trả {stance, axis_score, argument}.

⚡ Năng suấtEfficiency
✅ Chất lượngQuality
▤▤ chạy song song (thread pool)
→

③ Consolidate LLM

Tổng hợp verdict → final_score + rejected_reason + consolidator_note. Được phép bác dù judge đồng thuận.

💰 Chi phí mỗi candidate: 1 (extract) + N (judges) + 1 (consolidate) lời gọi ccs one -p — số candidate đã bị cap bởi --max-deepdive.
Sau debate: sort theo tổng final_score → giữ --top-candidates.

Synth LLM

uv run python scripts/synth.py --candidates … --top 3

Sinh skill draft + proposal cho top-N candidate được duyệt. Hai đường: A (skill-creator) → fallback B (template).

Mỗi candidate

  • Path A — ccs one -p dùng skill skill-creator ghi SKILL.md đúng spec Agent Skills (name + description + metadata)
  • Kiểm tra: nếu có SKILL.md → synth_path = "A"
  • Path B (fallback) — nếu A timeout/không ra file: ccs one -p nhỏ điền Jinja template (steps + 3 golden tests)
  • process_macro → tài liệu hoá flow 1→2→3→4; improvement_lesson → xoáy vào weak_points + improvement_notes
  • risk_flags write_action/deletes_files → buộc chèn bước confirm trước side-effect

Output

data/skills_<date>_proposal/
  • <skill-name>/SKILL.md (+ golden_tests.md, scripts/ nếu cần)
  • PROPOSAL.md — bản tóm tắt để người duyệt đọc
  • accept.py — installer tự chứa, đã bake tên skill
  • _proposal_meta.json — meta + synth_path từng skill
Skill drafts + PROPOSAL.md + accept.py▼

Accept USER

python data/skills_<date>_proposal/accept.py

Người dùng đọc PROPOSAL, chọn skill muốn giữ, cài vào kho skill cá nhân. Bước duyệt là của con người — không tự động.

Các bước

  • Đọc PROPOSAL.md, xem từng skill draft
  • Chạy accept.py — interactive (chọn số) hoặc accept.py 1 3 non-interactive
  • copytree <skill-name>/ → ~/.claude/skills/<skill-name>/ (bỏ qua nếu đã tồn tại)

Kết quả

~/.claude/skills/<skill-name>/
  • ✅ Skill được cài → active ở session Claude kế tiếp
  • Vòng lặp khép kín: hành vi của bạn → skill phục vụ lại chính bạn
